# 2. MVP 后端实现：认证 / 模型配置 / 会话消息

本文记录 Orbit MVP 后端第一阶段实现。

上一篇《MVP 数据库设计：用户 / 模型 / 会话》[^mvp-database-design]主要介绍 MVP 版本的数据库设计与代码实现，并记录后台初始化、数据库建表、ORM 与 DAO。

本文聚焦于业务代码的具体实现：包括认证、模型配置管理、会话管理、消息写入、模型调用、接口设计和错误落库。

当前版本的逻辑链是：

```text
用户注册 / 登录
  -> 创建模型配置
  -> 创建会话
  -> 发送用户消息
  -> 调用 OpenAI-compatible 或 Ollama
  -> 保存 assistant 消息
  -> 查询会话与消息历史
```

以下能力已经在数据结构中预留，属于下一阶段的实现范围：

```text
长会话自动摘要
LangGraph checkpointer / runtime 接入
更多 provider 适配
流式断点恢复
partial / cancelled 状态的完整恢复流程
```

## 1. 后端分层

当前后端采用 FastAPI + SQLAlchemy async + Alembic。

> [!TIP]
> 1. SQLAlchemy Async
>     
>     SQLAlchemy 的异步扩展，能够使用 async/await 语法来操作数据库
>
> 2. Alembic
>
>     数据库迁移工具，负责版本管理数据库表结构。

代码分层如下：

| 层级 | 目录 | 职责 |
| --- | --- | --- |
| API | `backend/app/api/v1` | 定义 HTTP 路由、鉴权依赖和响应模型 |
| Schemas | `backend/app/schemas` | 定义请求 / 响应结构，避免 ORM 直接暴露给前端 |
| Services | `backend/app/services` | 编排业务流程，例如注册、创建配置、发送消息 |
| Repositories | `backend/app/repositories` | 封装数据库读写 |
| Models | `backend/app/models` | SQLAlchemy ORM 模型 |
| Core | `backend/app/core` | 配置、安全、加密等基础能力 |
| DB | `backend/app/db` | 数据库 Session 与 Base |
| Migration | `backend/alembic` | 数据库迁移 |

其中，业务规则集中在 service 层，数据库查询集中在 repository 层。

## 2. 业务逻辑链路

### 2.1 数据库迁移与 ORM

MVP 的四张核心表已经通过 Alembic 迁移落地：

```text
users
llm_configs
conversations
messages
```

迁移文件：

```text
backend/alembic/versions/20260427_0001_create_mvp_tables.py
```

对应 ORM 模型：

```text
backend/app/models/user.py
backend/app/models/llm_config.py
backend/app/models/conversation.py
backend/app/models/message.py
```

当前实现相对于原始数据库设计补充了一个重要字段：

```text
users.password_hash
```

原因是 MVP 使用本地邮箱密码登录。后端只保存密码哈希，不保存明文密码。

### 2.2 认证实现

认证相关 API：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/api/v1/auth/register` | 注册用户，并直接返回访问令牌 |
| `POST` | `/api/v1/auth/login` | 邮箱密码登录 |
| `GET` | `/api/v1/auth/me` | 获取当前登录用户 |

核心代码：

```text
backend/app/api/v1/auth.py
backend/app/services/auth.py
backend/app/repositories/user.py
backend/app/core/security.py
```

注册流程：

```text
接收 email / password / display_name
  -> 校验邮箱是否已注册
  -> hash_password(password)
  -> 写入 users
  -> create_access_token(user.id)
  -> 返回 AuthToken
```

登录流程：

```text
接收 email / password
  -> 按邮箱查询用户
  -> 校验 password_hash
  -> 校验 is_enabled
  -> 返回 AuthToken
```

访问受保护接口时，前端需要携带：

```http
Authorization: Bearer <access_token>
```

### 2.3 模型配置实现

模型配置 API：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/v1/llm-configs` | 查询当前用户的模型配置 |
| `POST` | `/api/v1/llm-configs` | 创建模型配置 |
| `GET` | `/api/v1/llm-configs/{config_id}` | 查询单个模型配置 |
| `PATCH` | `/api/v1/llm-configs/{config_id}` | 更新模型配置 |
| `POST` | `/api/v1/llm-configs/{config_id}/default` | 设置默认模型配置 |
| `DELETE` | `/api/v1/llm-configs/{config_id}` | 软删除模型配置 |

核心代码：

```text
backend/app/api/v1/llm_configs.py
backend/app/services/llm_config.py
backend/app/repositories/llm_config.py
backend/app/core/crypto.py
```

创建模型配置时，服务层负责：

```text
校验同用户下 active name 唯一
  -> 判断是否第一条配置
  -> 必要时设置为默认配置
  -> 加密 api_key
  -> 写入 llm_configs
  -> 返回配置摘要
```

API Key 处理规则：

```text
入库：api_key -> encrypt_secret -> api_key_ciphertext
出库：不返回明文，不返回密文，只返回 has_api_key
调用模型：读取密文 -> decrypt_secret -> 注入 provider 请求
```

当前模型配置支持以下 provider：

```text
openai
openai_compatible
ollama
```

其中 `openai` 和 `openai_compatible` 走 Chat Completions 兼容接口；`ollama` 走本地 `/api/chat`。

### 2.4 会话实现

会话 API：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/v1/conversations` | 查询当前用户的会话列表 |
| `POST` | `/api/v1/conversations` | 创建会话 |
| `GET` | `/api/v1/conversations/{conversation_id}` | 查询会话详情 |
| `PATCH` | `/api/v1/conversations/{conversation_id}` | 更新会话标题、默认模型配置或 metadata |
| `DELETE` | `/api/v1/conversations/{conversation_id}` | 软删除会话 |

核心代码：

```text
backend/app/api/v1/conversations.py
backend/app/services/conversation.py
backend/app/repositories/conversation.py
```

创建会话时：

```text
接收 title / llm_config_id / chat_mode / metadata
  -> 如果未传 llm_config_id，尝试读取当前用户默认模型配置
  -> 校验指定模型配置属于当前用户
  -> 写入 conversations
  -> 数据库生成 thread_id
  -> 返回 ConversationRead
```

当前允许创建没有模型配置的会话。

这样做是为了让用户可以先建立会话壳，再去补模型配置。但真正发送消息调用模型时，如果仍然没有可用模型配置，会返回业务错误：

```text
请先创建并启用一个默认模型配置
```

### 2.5 消息与一问一答链路

消息 API：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/v1/conversations/{conversation_id}/messages` | 查询会话消息，按 `sequence_no` 升序 |
| `POST` | `/api/v1/conversations/{conversation_id}/messages` | 写入用户消息并生成 assistant 回复 |

发送消息的核心流程：

```text
校验 conversation 属于当前用户
  -> 确定 llm_config_id
  -> 校验模型配置存在且启用
  -> 生成下一条 sequence_no
  -> 写入 user message，status = completed
  -> 读取当前历史消息
  -> 写入 assistant placeholder，status = streaming
  -> 更新 conversations.updated_at
  -> commit
  -> 调用模型服务
  -> 成功：assistant status = completed
  -> 失败：assistant status = failed，response_metadata.error = 错误信息
  -> 再次更新 conversations.updated_at
  -> commit
```

返回结构是本轮交互的两条消息：

```json
{
  "user_message": {},
  "assistant_message": {}
}
```

`messages.sequence_no` 当前由：

```text
max(sequence_no) + 1
```

生成。这个策略对 MVP 的单会话单请求足够直接；如果后续支持同一会话并发发送消息，需要升级为行锁、独立计数器或数据库侧序列策略。

### 2.6 模型调用实现

模型调用统一由：`backend/app/services/llm_client.py` 负责。

调用入口： `LLMClient.generate(config, messages, summary)`。

上下文组装规则：

```text
如果 conversation.summary 存在
  -> 作为 system message 放入上下文

遍历历史 messages
  -> 只使用 status in completed / partial 的消息
  -> 只使用 role in system / user / assistant / tool 的消息
  -> 跳过空 content
```

OpenAI-compatible 请求：

```text
POST {base_url}/chat/completions
Authorization: Bearer <api_key>
body:
  model
  messages
  provider_options
  extra_body
```

Ollama 请求：

```text
POST {base_url}/api/chat
body:
  model
  messages
  stream = false
  options = provider_options
```

模型调用成功后，assistant 消息会保存：

```text
content
token_usage
response_metadata
provider
model
llm_config_id
```

模型调用失败时，不删除 assistant 占位消息，而是将其更新为：

```text
status = failed
response_metadata = {"error": "..."}
```

这样前端可以稳定展示失败状态，并在后续实现重试能力。

## 3. 关键代码

这一节记录业务层和接口层的关键代码。数据库初始化、建表、ORM 和 Repository / DAO 已经在上一篇文档的 `4. 关键代码` 中展开，本文只关注“认证 / 模型配置 / 会话消息”这条业务链路。

### 3.1 认证接口

认证接口定义在 `backend/app/api/v1/auth.py`。注册成功后直接返回访问令牌，前端可以立刻进入登录态。

```python
# backend/app/api/v1/auth.py
router = APIRouter(prefix="/auth", tags=["auth"])


@router.post("/register", response_model=AuthToken, status_code=201)
async def register(
    payload: UserCreate,
    session: Annotated[AsyncSession, Depends(get_db_session)],
) -> AuthToken:
    # 注册成功后直接返回访问令牌，前端可以立刻进入已登录状态。
    return await AuthService(session).register(payload)


@router.post("/login", response_model=AuthToken)
async def login(
    payload: UserLogin,
    session: Annotated[AsyncSession, Depends(get_db_session)],
) -> AuthToken:
    # 登录只校验邮箱密码，不在响应中暴露 password_hash 等内部字段。
    return await AuthService(session).login(payload)


@router.get("/me", response_model=UserRead)
async def get_me(current_user: Annotated[User, Depends(get_current_user)]) -> User:
    # 返回当前登录用户，供前端刷新页面后恢复登录态。
    return current_user
```

认证服务负责校验邮箱、处理密码哈希并签发 token。

```python
# backend/app/services/auth.py
class AuthService:
    # 认证服务只负责业务编排：校验用户、处理密码、签发访问令牌。
    def __init__(self, session: AsyncSession) -> None:
        self.session = session
        self.users = UserRepository(session)

    async def register(self, payload: UserCreate) -> AuthToken:
        # 注册前先按邮箱查重；邮箱唯一性最终仍由数据库索引兜底。
        existing_user = await self.users.get_by_email(str(payload.email))
        if existing_user is not None:
            raise HTTPException(
                status_code=status.HTTP_409_CONFLICT,
                detail="该邮箱已经注册",
            )

        user = await self.users.create(
            email=str(payload.email),
            password_hash=hash_password(payload.password),
            display_name=payload.display_name,
        )
        await self.session.commit()
        return self._build_token(user)

    async def login(self, payload: UserLogin) -> AuthToken:
        # 登录失败统一返回同一错误，避免暴露邮箱是否存在。
        user = await self.users.get_by_email(str(payload.email))
        if user is None or not verify_password(payload.password, user.password_hash):
            raise HTTPException(
                status_code=status.HTTP_401_UNAUTHORIZED,
                detail="邮箱或密码错误",
            )
        if not user.is_enabled:
            raise HTTPException(
                status_code=status.HTTP_403_FORBIDDEN,
                detail="账号已被禁用",
            )

        return self._build_token(user)
```

### 3.2 模型配置接口

模型配置接口统一挂在 `/llm-configs` 下。所有操作都依赖 `get_current_user`，因此天然按当前用户隔离。

```python
# backend/app/api/v1/llm_configs.py
router = APIRouter(prefix="/llm-configs", tags=["llm-configs"])


@router.get("", response_model=list[LLMConfigRead])
async def list_llm_configs(
    current_user: Annotated[User, Depends(get_current_user)],
    session: Annotated[AsyncSession, Depends(get_db_session)],
) -> list[LLMConfigRead]:
    # 返回当前用户自己的模型配置列表，API Key 只返回 has_api_key。
    return await LLMConfigService(session).list_configs(current_user.id)


@router.post("", response_model=LLMConfigRead, status_code=status.HTTP_201_CREATED)
async def create_llm_config(
    payload: LLMConfigCreate,
    current_user: Annotated[User, Depends(get_current_user)],
    session: Annotated[AsyncSession, Depends(get_db_session)],
) -> LLMConfigRead:
    # 创建配置时服务层会负责加密 API Key 和维护默认配置唯一性。
    return await LLMConfigService(session).create_config(user_id=current_user.id, payload=payload)
```

服务层处理默认配置、名称唯一性和 API Key 加密。

```python
# backend/app/services/llm_config.py
class LLMConfigService:
    # 模型配置服务负责用户维度的配置隔离、默认配置切换和 API Key 加密。
    def __init__(self, session: AsyncSession) -> None:
        self.session = session
        self.configs = LLMConfigRepository(session)

    async def create_config(self, *, user_id: UUID, payload: LLMConfigCreate) -> LLMConfigRead:
        # 同一用户下活跃配置名称保持唯一，方便前端做展示和切换。
        await self._ensure_name_available(user_id=user_id, name=payload.name)

        # 第一条模型配置自动成为默认配置，保证创建会话时有可用兜底。
        is_first_config = await self.configs.count_active(user_id) == 0
        is_default = payload.is_default or is_first_config
        if is_default:
            await self.configs.unset_defaults(user_id)

        # API Key 在进入数据库前加密；后续响应永远不返回明文或密文。
        config = await self.configs.create(
            user_id=user_id,
            name=payload.name,
            provider=payload.provider.strip(),
            model=payload.model.strip(),
            base_url=payload.base_url,
            api_key_ciphertext=encrypt_secret(payload.api_key),
            provider_options=payload.provider_options,
            is_default=is_default,
        )
        await self.session.commit()
        return self._to_read(config)
```

### 3.4 创建会话

创建会话时可以显式指定模型配置；如果没有指定，则尝试使用当前用户默认配置。

```python
# backend/app/services/conversation.py
async def create_conversation(
    self,
    *,
    user_id: UUID,
    payload: ConversationCreate,
) -> ConversationRead:
    # 创建会话时如果前端没指定模型配置，就尝试使用当前用户的默认配置。
    llm_config_id = payload.llm_config_id
    if llm_config_id is None:
        llm_config_id = await self._get_default_llm_config_id(user_id)
    elif await self.llm_configs.get_active(user_id=user_id, config_id=llm_config_id) is None:
        # 指定的模型配置必须属于当前用户，不能跨用户引用。
        raise HTTPException(status_code=status.HTTP_404_NOT_FOUND, detail="模型配置不存在")

    conversation = await self.conversations.create(
        user_id=user_id,
        title=payload.title,
        llm_config_id=llm_config_id,
        chat_mode=payload.chat_mode,
        metadata=payload.metadata,
    )
    await self.session.commit()
    return ConversationRead.model_validate(conversation)
```

### 3.5 发送消息与 assistant 落库

发送消息是当前后端最核心的一条链路：先写入用户消息和 assistant 占位消息，再调用模型，最后把 assistant 消息更新为 `completed` 或 `failed`。

```python
# backend/app/services/conversation.py
async def create_user_message(
    self,
    *,
    user_id: UUID,
    conversation_id: UUID,
    content: str,
) -> MessageExchangeRead:
    # 本接口完成“一问一答”：写入用户消息、创建 assistant 占位、调用模型并落库结果。
    conversation = await self._get_owned_conversation(
        user_id=user_id,
        conversation_id=conversation_id,
    )
    llm_config_id = conversation.llm_config_id or await self._get_default_llm_config_id(user_id)
    if llm_config_id is None:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="请先创建并启用一个默认模型配置",
        )

    llm_config = await self.llm_configs.get_active(user_id=user_id, config_id=llm_config_id)
    if llm_config is None or not llm_config.is_enabled:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="当前会话的模型配置不可用",
        )

    user_message = await self.messages.create_user_message(
        conversation_id=conversation_id,
        content=content,
    )
    history_messages = await self.messages.list_by_conversation(conversation_id)
    assistant_message = await self.messages.create_assistant_placeholder(
        conversation_id=conversation_id,
        llm_config_id=llm_config.id,
        provider=llm_config.provider,
        model=llm_config.model,
    )
    await self.conversations.touch(conversation_id)
    await self.session.commit()

    try:
        completion = await self.llm_client.generate(
            config=llm_config,
            messages=history_messages,
            summary=conversation.summary,
        )
    except LLMClientError as exc:
        assistant_message = await self.messages.fail_assistant_message(
            message=assistant_message,
            error=str(exc),
        )
        await self.conversations.touch(conversation_id)
        await self.session.commit()
        return MessageExchangeRead(
            user_message=MessageRead.model_validate(user_message),
            assistant_message=MessageRead.model_validate(assistant_message),
        )

    assistant_message = await self.messages.complete_assistant_message(
        message=assistant_message,
        content=completion.content,
        token_usage=completion.token_usage,
        response_metadata=completion.response_metadata,
    )
    await self.conversations.touch(conversation_id)
    await self.session.commit()
    return MessageExchangeRead(
        user_message=MessageRead.model_validate(user_message),
        assistant_message=MessageRead.model_validate(assistant_message),
    )
```

### 3.6 模型调用适配

模型调用集中在 `LLMClient` 中。Service 层不直接关心 OpenAI-compatible 或 Ollama 的请求细节，只接收统一的 `LLMCompletion`。

```python
# backend/app/services/llm_client.py
class LLMClient:
    # MVP 先支持 OpenAI Chat Completions 兼容协议和 Ollama，本地结构保留扩展点。
    async def generate(
        self,
        *,
        config: LLMConfig,
        messages: list[Message],
        summary: str | None = None,
    ) -> LLMCompletion:
        provider = config.provider.strip().lower()
        chat_messages = self._build_chat_messages(messages=messages, summary=summary)

        if provider in {"openai", "openai_compatible"}:
            return await self._generate_openai_compatible(
                config=config,
                messages=chat_messages,
            )
        if provider == "ollama":
            return await self._generate_ollama(config=config, messages=chat_messages)

        raise LLMClientError(f"暂不支持的模型供应商：{config.provider}")
```

上下文组装会过滤掉尚未完成的消息，避免把 `streaming` 或空内容传给上游模型。

```python
# backend/app/services/llm_client.py
def _build_chat_messages(
    self,
    *,
    messages: list[Message],
    summary: str | None,
) -> list[dict[str, str]]:
    chat_messages: list[dict[str, str]] = []
    if summary:
        # 摘要作为 system 上下文注入，不改写原始 messages 事实源。
        chat_messages.append(
            {
                "role": "system",
                "content": f"以下是此前对话摘要，请在后续回复中作为上下文参考：\n{summary}",
            }
        )

    for message in messages:
        if message.status not in {"completed", "partial"}:
            continue
        if message.role not in {"system", "user", "assistant", "tool"}:
            continue
        if not message.content:
            continue
        chat_messages.append({"role": message.role, "content": message.content})

    if not chat_messages:
        raise LLMClientError("没有可用于模型调用的消息上下文")
    return chat_messages
```

OpenAI-compatible provider 通过 Chat Completions 协议调用。

```python
# backend/app/services/llm_client.py
async def _generate_openai_compatible(
    self,
    *,
    config: LLMConfig,
    messages: list[dict[str, str]],
) -> LLMCompletion:
    api_key = decrypt_secret(config.api_key_ciphertext)
    if not api_key:
        raise LLMClientError("当前模型配置缺少 API Key")

    options = dict(config.provider_options or {})
    timeout = float(options.pop("timeout", 60))
    extra_body = options.pop("extra_body", None) or {}
    if not isinstance(extra_body, dict):
        raise LLMClientError("provider_options.extra_body 必须是对象")
    base_url = (config.base_url or "https://api.openai.com/v1").rstrip("/")
    payload = {
        **options,
        **extra_body,
        "model": config.model,
        "messages": messages,
    }

    try:
        async with httpx.AsyncClient(timeout=timeout) as client:
            response = await client.post(
                f"{base_url}/chat/completions",
                headers={"Authorization": f"Bearer {api_key}"},
                json=payload,
            )
            response.raise_for_status()
    except httpx.HTTPStatusError as exc:
        detail = self._extract_error_detail(exc.response)
        raise LLMClientError(f"模型服务返回错误：{detail}") from exc
    except httpx.HTTPError as exc:
        raise LLMClientError(f"模型服务请求失败：{exc}") from exc

    data = response.json()
    content = data.get("choices", [{}])[0].get("message", {}).get("content")
    if not isinstance(content, str) or content == "":
        raise LLMClientError("模型服务没有返回 assistant 内容")

    return LLMCompletion(
        content=content,
        token_usage=data.get("usage") or {},
        response_metadata={
            "provider": config.provider,
            "finish_reason": data.get("choices", [{}])[0].get("finish_reason"),
            "raw_id": data.get("id"),
        },
    )
```

```python
# backend/app/services/llm_client.py
async def _generate_ollama(
    self,
    *,
    config: LLMConfig,
    messages: list[dict[str, str]],
) -> LLMCompletion:
    options = dict(config.provider_options or {})
    timeout = float(options.pop("timeout", 120))
    base_url = (config.base_url or "http://127.0.0.1:11434").rstrip("/")
    payload = {
        "model": config.model,
        "messages": messages,
        "stream": False,
        "options": options,
    }

    try:
        async with httpx.AsyncClient(timeout=timeout) as client:
            response = await client.post(f"{base_url}/api/chat", json=payload)
            response.raise_for_status()
    except httpx.HTTPStatusError as exc:
        detail = self._extract_error_detail(exc.response)
        raise LLMClientError(f"Ollama 返回错误：{detail}") from exc
    except httpx.HTTPError as exc:
        raise LLMClientError(f"Ollama 请求失败：{exc}") from exc

    data = response.json()
    content = (data.get("message") or {}).get("content")
    if not isinstance(content, str) or content == "":
        raise LLMClientError("Ollama 没有返回 assistant 内容")

    return LLMCompletion(
        content=content,
        token_usage={
            "prompt_eval_count": data.get("prompt_eval_count"),
            "eval_count": data.get("eval_count"),
        },
        response_metadata={
            "provider": config.provider,
            "done": data.get("done"),
            "total_duration": data.get("total_duration"),
        },
    )
```
