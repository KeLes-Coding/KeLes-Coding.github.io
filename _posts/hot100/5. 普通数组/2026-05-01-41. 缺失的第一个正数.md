# 1. hot100-41. 缺失的第一个正数

[题目链接](https://leetcode.cn/problems/first-missing-positive/description/?envType=study-plan-v2&envId=top-100-liked)

给你一个未排序的整数数组 `nums` ，请你找出其中没有出现的最小的正整数。

请你实现时间复杂度为 `O(n)` 并且只使用常数级别额外空间的解决方案。

**示例 1：**

> **输入：** nums = [1,2,0]
> 
> **输出：** 3
> 
> **解释：** 范围 [1,2] 中的数字都在数组中。

**示例 2：**

> **输入：** nums = [3,4,-1,1]
> 
> **输出：** 2
> 
> **解释：** 1 在数组中，但 2 没有。

**示例 3：**

> **输入：** nums = [7,8,9,11,12]
> 
> **输出：** 1
> 
> **解释：** 最小的正数 1 没有出现。

**提示：**
- $1 <= nums.length <= 10^5$
- $-2^{31} <= nums[i] <= 2^{31} - 1$
---

## 思路

题目要求找到数组中缺失的第一个正数。对于长度为 `n` 的数组来说，答案一定在 `1 ~ n + 1` 之间。

因此，我们可以尝试把每个合法的正整数 x 放到它应该在的位置：下标 `x - 1`。

```cpp
数字 1 放在 nums[0]
数字 2 放在 nums[1]
数字 3 放在 nums[2]
...
数字 x 放在 nums[x - 1]
```

遍历数组时，如果当前数字 `nums[i]` 在 `1 ~ n` 范围内，并且它还没有放到正确位置，就不断交换它到对应位置。

交换完成后，再从左到右检查数组：

如果 `nums[i] != i + 1`，说明 `i + 1` 这个正数缺失；
如果所有位置都正确，说明 `1 ~ n` 都存在，答案就是 `n + 1`。

---

## 题解

```cpp
class Solution {
public:
    int firstMissingPositive(vector<int>& nums) {
        int n = nums.size();

        for (int i = 0; i < n; i++) {
            while (nums[i] >= 1 && nums[i] <= n && nums[nums[i] - 1] != nums[i]) {
                swap(nums[i], nums[nums[i] - 1]);
            }
        }

        for (int i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                return i + 1;
            }
        }

        return n + 1;
    }
};
```

## 复杂度分析

时间复杂度：`O(n)`。

空间复杂度：`O(1)`。
