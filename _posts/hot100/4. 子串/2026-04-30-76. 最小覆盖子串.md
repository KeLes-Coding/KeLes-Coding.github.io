# 1. hot100-76. 最小覆盖子串

[题目链接](https://leetcode.cn/problems/minimum-window-substring/description/?envType=study-plan-v2&envId=top-100-liked)

给定两个字符串 `s` 和 `t`，长度分别是 `m` 和 `n`，返回 s 中的 **最短窗口 子串**，使得该子串包含 `t` 中的每一个字符（**包括重复字符**）。如果没有这样的子串，返回空字符串 `""`。

测试用例保证答案唯一。

 

**示例 1：**

> **输入：** s = "ADOBECODEBANC", t = "ABC"
> 
> **输出：** "BANC"
> 
> **解释：** 最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。

**示例 2：**

> **输入：** s = "a", t = "a"
> 
> **输出：** "a"
> 
> **解释：** 整个字符串 s 是最小覆盖子串。

**示例 3:**

> **输入:** s = "a", t = "aa"
> 
> **输出:** ""
> 
> **解释:** t 中两个字符 'a' 均应包含在 s 的子串中，
因此没有符合条件的子字符串，返回空字符串。

**提示：**
- `m == s.length`
- `n == t.length`
- $1 <= m, n <= 10^5$
- `s` 和 `t` 由英文字母组成

**进阶：** 你能设计一个在 `O(m + n)` 时间内解决此问题的算法吗？

---

## 思路
本体要求在 `s` 中找到一个最短子串，使其包含 `t` 中的所有字符，同时字符数量也满足要求。

同一专题中，《滑动窗口最大值》[^sliding-window-maximum]更强调窗口内候选值的单调维护；《和为 K 的子数组》[^subarray-sum-k]则更适合用来对比前缀和与哈希计数的区间问题建模。

使用滑动窗口 `[left, right]` 来维护区间字符，同时 `right - left + 1` 计算滑动窗口长度。

维护两个 `map` `need` 与`window` ， 结构均为 `unordered_map<char, int>`。

其中 `need` 记录 `t` 中每个字符需要出现的次数，`window` 记录当前窗口中每个字符出现的次数。

同时使用两个变量：

- `t_require`：表示一共有多少种字符需要满足数量要求
- `formed`：表示当前窗口中已经满足数量要求的字符种类数

当 `formed == t_require` 时，说明当前窗口已经覆盖了 `t`，此时尝试移动 `left` 缩小窗口，并在缩小过程中更新最短答案。

如果移出某个字符后，导致它在窗口中的数量少于 `need` 中要求的数量，就说明当前窗口不再合法，需要停止收缩，继续右移 `right` 扩大窗口。

### 遍历过程
1. 右指针 `right` 不断向右移动，把字符加入窗口。
2. 如果某个字符的出现次数刚好达到 `need` 的要求，则 `formed++`。
3. 当 `formed == t_require` 时，说明窗口已经满足条件。
4. 此时不断右移 `left`，尝试缩小窗口，并更新最短答案。
5. 如果移出的字符导致窗口不再满足要求，则 `formed--`，停止收缩。

---

## 题解

```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        unordered_map<char, int> need;
        unordered_map<char, int> window;

        for (char c : t) {
            need[c]++;
        }

        int ans = INT_MAX;
        int start = 0;
        int t_require = need.size();
        int formed = 0;

        int left = 0, right = 0;
        while(right < s.size()) {
            char c = s[right];
            window[c]++;

            if (need.count(c) && window[c] == need[c]) {
                formed++;
            }

            while (left <= right && formed == t_require) {
                if (right - left + 1 < ans) {
                    ans = right - left + 1;
                    start = left;
                }

                char d = s[left];
                window[d]--;

                if (need.count(d) && window[d] < need[d]) {
                    formed--;
                }
                left++;
            }
            right++;
        }

        return ans == INT_MAX ? "" : s.substr(start, ans);
    }
};
```

## 复杂度分析
设 `s` 长度为 `n`，`t` 长度为 `m`。

时间复杂度 `O(n+m)`

空间复杂度取决于字符出现数量。小于等于 52。
