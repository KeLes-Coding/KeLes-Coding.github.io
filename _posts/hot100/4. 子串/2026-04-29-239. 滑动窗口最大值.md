# 1. hot100-239. 滑动窗口最大值

[题目链接](https://leetcode.cn/problems/sliding-window-maximum/?envType=study-plan-v2&envId=top-100-liked)

给你一个整数数组 `nums`，有一个大小为 `k` 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 `k` 个数字。滑动窗口每次只向右移动一位。

返回 滑动窗口中的最大值 。

 

**示例 1：**

```
输入：nums = [1,3,-1,-3,5,3,6,7], k = 3
输出：[3,3,5,5,6,7]
解释：
滑动窗口的位置                最大值
---------------               -----
[1  3  -1] -3  5  3  6  7      3
 1 [3  -1  -3] 5  3  6  7      3
 1  3 [-1  -3  5] 3  6  7      5
 1  3  -1 [-3  5  3] 6  7      5
 1  3  -1  -3 [5  3  6] 7      6
 1  3  -1  -3  5 [3  6  7]     7
```

**示例 2：**

```
输入：nums = [1], k = 1
输出：[1]
```

 

提示：
- $1 <= nums.length <= 10^5$
- $-10^4 <= nums[i] <= 10^4$
- $1 <= k <= nums.length$

---

## 思路

这题的目标是在某一个长度为 k 的滑动窗口中，找到当前窗口的最大值。

一个基本的思路是遍历整个窗口求最大值，由于 k 不是常数，时间复杂度会达到 `O(nk)`。一个优化思路是，维护一个 **单调队列**，存储的是滑动窗口的下标，排列则按照下标对应的原数值从大到小排列。

即：
```text
1. 队首下标必须在当前窗口范围内；
2. 队列中对应的元素值从队首到队尾单调递减。
```
此时，队首元素对应的值就是当前窗口的最大值。

### 遍历过程：
1. 如果队首下标已经滑出窗口，即 `q.front() <= i - k`，将其弹出；
2. 维护单调递减队列：如果队尾元素小于当前元素 `nums[i]`，说明它之后不可能成为窗口最大值，将其弹出；
3. 将当前下标 `i` 加入队尾；
4. 当 `i >= k - 1` 时，说明窗口已经形成，此时将 `nums[q.front()]` 加入答案。

---

## 题解

```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        deque<int> q;
        vector<int> ans;

        for (int i = 0; i < nums.size(); i++) {
            // 如果队首元素已经不在当前窗口里，就把它删掉。
            while (!q.empty() && q.front() <= i - k) {
                q.pop_front();
            }

            while (!q.empty() && nums[q.back()] < nums[i]) {
                q.pop_back();
            }
            q.push_back(i);

            if (i >= k - 1) {
                ans.push_back(nums[q.front()]);
            }
        }

        return ans;
    }
};
```

## 复杂度分析
时间复杂度：`O(n)`。

空间复杂度：`O(k)`。
